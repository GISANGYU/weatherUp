FINAL PROJECT PLAN

Weather Mode

날씨 데이터를 기반으로 웹사이트의 테마와 패션, 음식, 액티비티 추천이 함께 변화하는 라이프스타일 큐레이션 웹 서비스

“날씨를 UI 모드로 활용해 하루의 라이프스타일을 큐레이션하는 인터랙티브 웹 서비스”

1. 프로젝트 개요

프로젝트명

Weather Mode

한 줄 소개

사용자의 지역과 선택한 날짜의 날씨 데이터를 기반으로 웹사이트의 테마와 패션, 음식, 액티비티 추천 콘텐츠가 함께 변화하는 라이프스타일 큐레이션 서비스

2. 프로젝트 목적

기존 날씨 서비스는 주로 기온과 날씨 상태를 전달하는 데 집중되어 있다. 하지만 실제 사용자의 하루는 날씨에 따라 옷차림, 식사, 활동 방식까지 크게 달라진다.

본 프로젝트는 날씨를 단순 정보가 아니라 사용자의 하루를 구성하는 기준 데이터로 활용하여, 더 실질적인 라이프스타일 큐레이션 경험을 제공하는 것을 목표로 한다.

3. 기획 의도

문제 정의

  • 기존 날씨 앱은 정보 제공 중심으로 끝나는 경우가 많다.
  • 날씨와 실제 행동 선택 사이의 연결이 약하다.
  • 사용자는 날씨를 본 뒤 다시 별도로 옷, 음식, 활동을 고민해야 한다.

해결 방향

  • 날씨를 기준으로 하루를 제안하는 구조 설계
  • 정보 제공을 넘어 행동과 감성까지 연결
  • UI 자체를 날씨 기반으로 변화시키는 시스템 도입
Weather-based Theme Switching Lifestyle Curation Interactive UI API-driven Content Component-based Design

4. 핵심 컨셉

서비스 개념

사용자가 상단의 7일 예보 중 특정 날짜를 선택하면, 해당 날짜의 날씨 상태에 따라 웹사이트 전체의 시각적 테마와 라이프스타일 추천 콘텐츠가 동시에 변화한다.

1

날씨 데이터 수집

현재 위치 또는 검색한 지역을 기준으로 날씨 데이터를 불러온다.

2

날짜 선택

사용자가 일주일 예보 중 원하는 날짜를 선택한다.

3

테마 전환

맑음, 흐림, 비, 눈 등 날씨 상태를 기준으로 웹사이트의 모드가 바뀐다.

4

콘텐츠 큐레이션

패션, 음식, 액티비티 섹션이 해당 날씨에 맞게 갱신된다.

5. 주요 기능

0. 오늘의 날씨

  • 사용자 위치 기반 현재 지역 날씨 표시
  • 위치 허용이 어려운 경우 지역 검색 기능 제공
  • 7일 예보 버튼 제공
  • 날짜 선택 시 전체 섹션 동기화

1. 패션 섹션

  • 오늘 뭐 입을지 제안하는 한 줄 문구
  • 추천 스타일 키워드 및 준비물 표시
  • 우산, 아우터, 선글라스 등 상황형 배지 제공

기능 계속

2. 음식 섹션

  • 날씨와 지역을 함께 고려한 음식 추천
  • 현재 위치 또는 검색 지역 기준 맛집 리스트 제공
  • 비 오는 날 파전, 추운 날 국물요리 등 직관적 큐레이션

3. 액티비티 섹션

  • 실내/실외 활동 제안
  • 날씨에 따른 콘텐츠 및 장소 추천
  • 한 줄 감성 카피를 통한 경험 강화

15. 차별성

  • 단순 날씨 정보 제공이 아닌 행동 기반 추천 제공
  • 날씨에 따라 UI 자체가 변하는 테마 시스템
  • 패션, 음식, 액티비티를 통합한 라이프스타일 큐레이션
  • 컴포넌트 중심 구조로 확장성 확보

16. 기대 효과

  • 데이터 기반 UI 설계 경험 확보
  • API 연동 능력과 컴포넌트 설계 능력 강화
  • 사용자 중심 UX 기획 능력 증명
  • 시각적 전환과 정보 구조를 동시에 보여주는 결과물 제작

17. 결론

Weather Mode는 날씨를 단순한 정보가 아니라 사용자의 하루를 구성하는 핵심 기준으로 바라보는 웹 서비스이다. 사용자는 날씨를 확인하는 데서 끝나지 않고, 그날의 스타일, 음식, 활동까지 한 번에 제안받을 수 있다.

이를 통해 본 프로젝트는 날씨 정보 + 시각적 테마 전환 + 라이프스타일 큐레이션을 결합한 경험 중심 인터페이스를 구현하고자 한다.

교수 발표용 핵심 문장: 날씨를 UI 모드로 활용해 하루의 라이프스타일을 큐레이션하는 인터랙티브 웹 서비스입니다.
